# 画布原点 / 默认画笔格点测试（256×256 逻辑网格）

## 你已测定的显示参数（720p 截图坐标系）

- 贴图绘制模式进入后，画布在截图里约为 **512×512** 的显示区域（你在外部工具里用 **锚点居中、中心 (640, 410)、边长 512** 对齐）。
- 游戏内**可移动/落笔的离散网格**为 **256×256**（显示为 2× 放大，一步对应显示上的约 2px）。
- **默认画笔**：画布 **512×512 的几何中心是格线十字交点**；进入绘图模式且未移动时，光标落在该交点 **右下** 那一格（相对交点：列 +1、行 +1）。在本仓库 `row`↓、`col`→ 约定下，该格为 **`(row0, col0) = (128, 128)`**（与「中心落在四格共用角上」时东南象限一致）。从默认位置回到逻辑 **(0,0)** 需 **向左 128 步、向上 128 步**（与游戏方向一致时）。若你仍想用撞边验证，应测得 **L = U = 128**。

逻辑坐标约定（与本仓库 `indices.bin` 一致）：

- `row`：0 在**上**，向下增大。  
- `col`：0 在**左**，向右增大。  
- 扁平索引：`i = row * 256 + col`。  
- `0xFF`：**NO_DRAW**。

## 方法一：撞边计数（推荐，纯手柄、无需导入图）

1. 进入贴图绘制，**不要移动**画笔，确认在默认中心。
2. 按住 **向左**，一格一格数，直到到达**最左列**（无法再左或明显顶到边界），记 **L**。
3. 回到绘制模式默认中心（退出重进最快）。
4. 再数 **向上** 到**最上行**，记 **U**。

解释：

- 若中心格列为 **127**（0 起始），则到左边界应 **127** 步（左侧还有 127 格）。  
- 若中心格列为 **128**，则 **128** 步。  
- 行方向同理。

若 L 与 U 不一致，可能是不对称 UI 或计数误差，多试一次或录屏慢放。

## 方法一 B：固件慢速单步（数格子更省力）

仓库 sketch：[../firmware/brush_grid_count_probe/brush_grid_count_probe.ino](../firmware/brush_grid_count_probe/brush_grid_count_probe.ino)

1. 打开该文件，在顶部 **只启用** `PROBE_LEFT_COLUMN 1` **或** `PROBE_UP_ROW 1`（另一个必须为 0）。  
2. 上传后插 **Switch**，进入贴图绘制默认中心。  
3. **约 5 秒**后开始动；之后每隔 **STEP_PAUSE_MS**（默认 1600ms）**点按一次** 左或上。  
4. **从第一次看到光标进新格**开始数 **1**，每再移动一格 **+1**，直到明显**撞边**不再移动；记下总数。  
5. **换方向**：改 `#define`、重新上传，再测另一轴。

**解释**：从默认格 `(row0, col0)` 到左边界需 **`col0` 次**「向左一格」；到上边界需 **`row0` 次**「向上一格」。若 256×256 且中心对称，常见 **`L = U = 127`** 或 **`128`**。

**若一步跳多格**：游戏可能用了加速或长按连发——先改用手动方法一，或把 `STEP_PAUSE_MS` 加大、并检查是否需在设置里关掉方向键连发；必要时再改用 `tiltLeftStick` 微调（另开固件实验）。

## 方法一 C：四角巡游（验证步进与 HOME 常数）

固件：[../firmware/canvas_corners_tour/canvas_corners_tour.ino](../firmware/canvas_corners_tour/canvas_corners_tour.ino)

- 假设默认格为 **`(128, 128)`**：先 **左 128、上 128** 到 **(0,0)**，再沿边 **右 255 → 下 255 → 左 255**；**每角按一次 A 落一点**（默认 `STAMP_DOT_AT_EACH_CORNER`），便于对照格线；不要污点则将该宏设为 **0**。
- **热插拔**时若出现「请在使用的手柄按下 L+R / 再按 A」，固件会在十字键动作前自动发送 **L+R** 与 **A**；若你已配对、插线不再弹窗，可将 `RUN_SWITCH_CONTROLLER_PAIRING` 设为 **0** 以免误触肩键。
- 若某一角固定差 1 格：先调 `STEP_GAP_MS`；仍不对再核对 `HOME_ROW0` / `HOME_COL0` 或游戏是否对十字键加速。

## 方法二：单点探针文件（配合未来自动机 / 或人工对照）

仓库脚本可生成 **仅含 1 个可绘制像素** 的 `256×256` PNG + `*_indices.bin`：

```bash
cd scripts/texture_prep
python generate_probe.py ^
  --palette-json ../../assets/generated/palette_default.json ^
  --out-prefix ../../assets/test/home_center_probe/probe_127_127 ^
  --pixel 127,127
```

将 `row,col` 换成 `127,128`、`128,127`、`128,128` 各生成一份。

- `--palette-index` 默认 **1**（非纯白，便于在白底上看点）；可改成任意 `0..83`。
- 输出：`probe_*.png`、`*_indices.bin`、`*_meta.json`、`*_sidecar.json`（探针说明）。

**人工对照思路**：若游戏允许把「目标图案」以某种方式叠在画布上，可用 PNG 对齐；否则仍以**方法一**为准。
